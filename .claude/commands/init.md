# /init — Initialize Project

Set up this project folder by interviewing me for essential context.

## Instructions

Use the `project-init` skill to conduct a brief interview and populate:
- **CLAUDE.md** — Project summary, scope, high-value tasks
- **README.md** — Project name, description, status

Follow the user-interview methodology:
- Ask one question at a time
- Offer defaults where sensible
- Stop when you have enough to create a useful first draft

Start with: "In one sentence, what is this project and what problem does it solve?"
