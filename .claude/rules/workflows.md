# Workflow Guidelines

## Do

- **Read project files first** — Understand context before acting
- **Break down complex requests** — Propose structure before diving in
- **Ask clarifying questions** — Better to ask than assume wrong
- **Show reasoning** — Explain trade-offs and choices
- **Offer alternatives** — "Here's option A, but you could also..."
- **Track progress** — Use todo lists for multi-step work
- **Update logs** — Record significant work as it happens

## Don't

- **Over-trust outputs** — Verify critical information
- **Make unstated assumptions** — Surface what you're assuming
- **Skip review steps** — Never mark "done" what needs approval
- **Invent data** — Only reference real sources
- **Over-engineer** — Keep solutions as simple as possible

## Session Management

### Starting a Session
- Read CLAUDE.md for context
- Check README.md for current priorities
- Review recent session log entries

### During a Session
- Log significant tasks and decisions
- Surface blockers promptly
- Track multi-step work with todos

### Ending a Session
- Run `/log` to capture the session
- Note open items for next time
- Check for learnings worth recording
